# Event Decision-Making Simulation

## Overview
This is an individual asynchronous simulation on event decision-making from planning to execution.

Students will:
- review the initial event scenario
- identify priorities, risks, and contingency plans
- respond to sudden incidents
- decide whether the event should proceed, be modified, postponed, or canceled
- complete a final reflection
- submit proof of completion

## Suggested Duration
40 minutes

## Student Instructions
1. Open the simulation page.
2. Complete the planning section.
3. Click **Start Simulation**.
4. Respond to all incident stages.
5. Complete the final reflection.
6. Take a screenshot of the final results page.
7. Download the decision log file.
8. Submit both as proof of completion.

## Submission Proof
- 1 screenshot of final results page
- 1 downloaded JSON decision log file

## AI Disclosure
If required by your instructor, include your AI disclosure in the space provided in the simulation.

## How to Run in GitHub
1. Upload `index.html`, `styles.css`, `script.js`, and `README.md` to your repository.
2. Go to **Settings** of the repository.
3. Click **Pages**.
4. Under **Build and deployment**, select:
   - Source: **Deploy from a branch**
   - Branch: **main**
   - Folder: **/root**
5. Save.
6. GitHub will generate a public link for the simulation.
